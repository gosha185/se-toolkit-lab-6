# План реализации Task 2: The Documentation Agent

## Обзор

В этой задаче мы превращаем простой CLI-чатбот в настоящего **агента** с инструментами. Агент сможет читать файлы проекта и навигировать по директориям, чтобы отвечать на вопросы по документации.

## Архитектура

### 1. Схемы инструментов (Tool Schemas)

Определим два инструмента как функции с явными параметрами:

#### `read_file(path: str) -> str`
- **Назначение:** Читает содержимое файла по относительному пути
- **Возвращает:** Строку с содержимым файла или сообщение об ошибке
- **Безопасность:** Проверка на выход за пределы проекта (запрет `../`)

#### `list_files(path: str) -> str`
- **Назначение:** Возвращает список файлов и директорий в указанной директории
- **Возвращает:** Строку с именами файлов, разделёнными новыми строками
- **Безопасность:** Проверка на выход за пределы проекта

**Формат для LLM:** Передадим схемы инструментов в параметре `tools` API запроса. LLM будет возвращать `tool_calls` в ответе.

### 2. Агентовый цикл (Agentic Loop)

Реализуем цикл типа ReAct (Reason + Act):

```
1. Отправляем вопрос пользователя + описание инструментов в LLM
2. Получаем ответ:
   - Если есть `tool_calls` → выполняем инструменты, добавляем результаты в историю
   - Если нет `tool_calls` → это финальный ответ, извлекаем answer + source
3. Повторяем шаг 1 с обновлённой историей (максимум 10 итераций)
4. Возвращаем JSON с answer, source, tool_calls
```

**Структура сообщений:**
```python
messages = [
    {"role": "system", "content": system_prompt},
    {"role": "user", "content": question}
]

# Цикл:
while tool_call_count < 10:
    response = call_llm(messages)
    if response has tool_calls:
        execute tools
        append tool results as {"role": "tool", ...}
    else:
        break  # финальный ответ
```

### 3. Безопасность путей (Path Security)

Защита от чтения файлов вне проекта:

```python
def is_safe_path(path: str) -> bool:
    # Нормализуем путь
    full_path = (PROJECT_ROOT / path).resolve()
    # Проверяем, что путь начинается с PROJECT_ROOT
    return full_path.is_relative_to(PROJECT_ROOT)
```

**Правила:**
- Запретить пути, содержащие `..`
- Запретить абсолютные пути
- После нормализации путь должен оставаться внутри проекта

### 4. Системный промпт

Промпт должен инструктировать LLM:

1. Использовать `list_files` для обнаружения файлов в wiki/
2. Использовать `read_file` для чтения содержимого
3. Извлекать источник (файл + секция) для поля `source`
4. Возвращать финальный ответ в JSON формате

**Пример:**
```
You are a documentation agent with access to the project wiki.
Use list_files to discover files, then read_file to find answers.
Always include the source reference (file path + section anchor).
Respond with JSON: {"answer": "...", "source": "wiki/file.md#section"}
```
